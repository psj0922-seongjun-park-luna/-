## ☕ Java의 기본 구동 원리: 컴파일과 실행 (2025년 3월 25일)

자바 프로그램을 실행하기 위해서는 **컴파일(Compile)**과 **실행(Execute)**이라는 두 가지 단계가 필요합니다.

### 1. javac.exe (Compiler)
- **역할:** 사용자가 작성한 소스 코드(`.java` 파일)를 컴퓨터가 이해할 수 있는 바이트코드(`.class` 파일)로 변환합니다.
- **특징:** 번역가와 같은 역할을 하며, 문법 오류를 체크합니다.
- **명령어:** `javac HelloWorld.java`

### 2. java.exe (Interpreter)
- **역할:** 컴파일된 바이트코드(`.class` 파일)를 실제로 한 줄씩 읽어서 실행합니다.
- **특징:** 통역사(Interpreter)와 같은 역할을 하며, 프로그램을 구동시킵니다.
- **명령어:** `java HelloWorld`

---
> **💡 한 줄 요약**
> `javac`는 기계가 읽을 수 있게 **번역**하는 명령어고, `java`는 번역된 내용을 **실행**하는 명령어입니다.
# 📖 Java의 기본 개념 및 구동 원리 정리

자바 개발의 핵심인 3가지 에디션과 압축 형식, 그리고 프로그램이 실행되는 내부 메커니즘을 정리했습니다.

---

## 1️⃣ 자바 개발의 3대 환경 (Java Editions)
개발 목적에 따라 에디션이 나뉘지만, **모든 환경의 기초는 Java SE(JDK)**입니다.

| 구분 | 에디션 명칭 | 주요 용도 | 비고 |
| :--- | :--- | :--- | :--- |
| **Java SE** | **Standard Edition** | 자바의 표준/기본 베이스 | **JDK 설치 필수** |
| **Java EE** | **Enterprise Edition** | 기업용 웹 개발 (Servlet/JSP) | 현 Jakarta EE |
| **Java ME** | **Micro Edition** | 모바일/가전용 소형 환경 | 임베디드 등 |

> **📌 핵심:** 무엇을 개발하든 표준 환경인 **SE(JDK)**가 반드시 먼저 구축되어야 합니다.

---

## 2️⃣ 자바 기반 압축 파일 형식 (Archive Types)
자바는 배포 효율을 위해 용도별로 파일을 압축하여 관리합니다.

1. **.jar (Java Archive)**
   - 컴파일된 `*.class` 파일들을 하나로 묶은 기본 압축 형식.
2. **.war (Web Archive)**
   - 웹 애플리케이션(Servlet, JSP 등) 구동에 필요한 파일들을 압축한 형식.
3. **.ear (Enterprise Archive)**
   - 가장 큰 단위로, `.jar`와 `.war`를 합쳐 기업용 시스템 전체를 압축한 형식.

---

## 3️⃣ 자바 프로그램 작성 및 실행 메커니즘
사람이 쓴 코드가 컴퓨터에서 실행되기까지는 **컴파일**과 **실행**이라는 두 단계가 필요합니다.

### ① 소스 코드 작성 (Source Code)
- 파일명: `Test.java`
- 특징: 사람이 이해할 수 있는 영어 형태 (ex: `public class Test { ... }`)

### ② 컴파일 작업 (Compile)
- **명령어:** `javac Test.java`
- **도구:** 컴파일러 (`javac.exe`)
- **결과:** 컴퓨터(JVM)가 이해할 수 있는 **바이트 코드(`Test.class`)** 생성.

### ③ 실행 과정 (Execute)
- **명령어:** `java Test` (확장자 제외)
- **도구:** 인터프리터 (`java.exe`)
- **결과:** `.class` 파일을 한 줄씩 읽어 실제 프로그램 구동.

## 1. 개발 도구 (Eclipse) 설정
- **작업 디렉터리:** `C:\cloud_engineering\java21_study\eclipse\workspace`
- **실행 파일:** `eclipse.exe`
- **환경 확인:** `Window > Preferences`에서 컴파일러 및 JRE 설정 확인.

## 2. 프로젝트 및 클래스 생성 규칙
- **Java Project:** `File > New > Java Project` (※ Module 체크 해제 필수)
- **Class 생성:** `Name`은 파일명과 일치해야 하며, `public static void main` 체크.
- **실행:** `Run As > Java Application` (콘솔 출력 확인)

## 3. 자바 식별자 (Identifier)
1. **시스템 정의 식별자 (예약어):** 시스템이 미리 정의한 단어 (예: `public`, `class`, `static`). 수정 시 에러 발생.
2. **사용자 정의 식별자:** 개발자가 명명하는 클래스, 변수, 메서드명.
   - 클래스명은 반드시 **대문자**로 시작.
   - 의미 있는 명사형 권장 및 파일명과 일치 필수.
   - 특수문자는 `_`, `$`만 허용.

## 4. main 메서드 및 API
- **main:** 프로그램의 시작점(Starting Point). 어플리케이션당 단 하나 존재.
- **Java API:** 개발을 돕는 클래스 묶음 (`.jar` 형태). `System`, `String` 등이 대표적임.

## 5. 데이터 출력 (Standard Output)
1. `System.out.println()`: 출력 후 줄 바꿈.
2. `System.out.print()`: 줄 바꿈 없이 출력.
3. `System.out.printf()`: 포맷 지정 출력 (형식과 값의 일치 필요).


# 🚀 [Study] Git 기초 개념 및 필수 명령어 정리

### 1. Git이란 무엇인가?
* **정의:** 코드의 과거와 현재를 기록하는 **'타임머신'** (분산 버전 관리 시스템)
* **핵심 용도:** 1. 실수했을 때 과거의 특정 시점으로 되돌리기 위해
    2. 여러 사람이 하나의 프로젝트를 안전하게 협업하기 위해
* **제작자:** 리누스 토르발스 (Linux 커널 개발자)

---

### 2. Git의 3단계 작업 흐름
파일이 저장소에 기록되기까지는 반드시 아래 3단계를 거칩니다.



1. **Working Directory (내 방):** 현재 내가 코드를 수정하고 있는 실제 폴더
2. **Staging Area (현관문):** 버전을 만들기 위해 파일들을 잠시 모아두는 곳 (`git add`)
3. **Local Repository (창고):** 버전이 영구적으로 저장되는 저장소 (`git commit`)

---

### 3. 필수 명령어 치트 시트 (Cheat Sheet)

| 구분 | 명령어 | 설명 및 비유 |
| :--- | :--- | :--- |
| **시작** | `git init` | 현재 폴더를 Git 저장소로 **초기화** (관리 시작) |
| **준비** | `git add <파일명>` | 수정된 파일을 **장바구니(Staging)**에 담기 |
| **기록** | `git commit -m "메시지"` | 장바구니에 담긴 파일들을 **하나의 버전**으로 박제 |
| **확인** | `git log` | 지금까지 찍은 **버전 사진첩(이력)** 보기 |
| **비교** | `git diff` | 이전 버전과 지금 내 코드가 **어디가 다른지** 확인 |
| **복구** | `git reset` / `git revert` | 잘못했을 때 **과거의 시점**으로 돌아가기 |
| **합치기** | `git merge` | 각자 작업한 **나뭇가지(Branch)를 하나로** 합치기 |
